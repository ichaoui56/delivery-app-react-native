
# Deployment Guide

This guide provides instructions on how to build and deploy the Delivery Man mobile application to the Google Play Store and Apple App Store.

## 1. Prerequisites

- Node.js and npm installed
- Expo CLI installed (`npm install -g expo-cli`)
- An Expo account
- A Google Play Developer account
- An Apple Developer account

## 2. Generating a Release Build

### Android (APK / AAB)

1.  **Configure the App:**
    - Open `app.json` and ensure the `android.package` is unique.
    - Set the `version` and `android.versionCode`.

2.  **Start the Build:**
    - Run the following command to build an Android App Bundle (AAB):
      ```bash
      expo build:android -t aab
      ```
    - To build an APK, run:
      ```bash
      expo build:android -t apk
      ```

3.  **Download the Build:**
    - Once the build is complete, you will get a URL to download the AAB/APK file.

### iOS (IPA)

1.  **Configure the App:**
    - Open `app.json` and ensure the `ios.bundleIdentifier` is unique.
    - Set the `version` and `ios.buildNumber`.

2.  **Start the Build:**
    - Run the following command:
      ```bash
      expo build:ios
      ```

3.  **Download the Build:**
    - After the build is finished, download the IPA file from the provided URL.

## 3. Publishing to the Google Play Store

1.  **Create a New App:**
    - In the Google Play Console, click "Create app" and fill in the app details.

2.  **Upload the AAB:**
    - Go to "Production" > "Create new release".
    - Upload the AAB file generated by Expo.

3.  **App Content and Store Listing:**
    - Fill in all the required information, including:
      - App content (privacy policy, ads, etc.)
      - Store listing (short and long descriptions, screenshots, etc.)

4.  **Rollout the Release:**
    - Review and roll out the release.

## 4. Publishing to the Apple App Store

1.  **Create an App ID:**
    - In your Apple Developer account, create a unique App ID for your application.

2.  **Create a Provisioning Profile:**
    - Create a provisioning profile to allow your app to be installed on devices.

3.  **Upload to App Store Connect:**
    - Use the Expo Application Uploader or Transporter to upload the IPA file to App Store Connect.
      ```bash
      expo upload:ios
      ```

4.  **Submit for Review:**
    - In App Store Connect, fill in all the required metadata and submit the app for review.
    - You can use TestFlight to test your app before submitting it for review.
